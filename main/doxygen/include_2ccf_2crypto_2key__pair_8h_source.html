64 // Note about the signed_by_issuer parameter to sign_csr: when issuing a new
65 // certificate for an old subject, which does not exist anymore, we cannot
66 // sign the CSR with that old subject's private key. Instead, the issuer
67 // signs the CSR itself, which is slightly unusal. Instead, we could also
68 // ask the subject to produce a CSR right after it becomes alive and keep it
69 // around until we need it, but those complications are not stricly
70 // necessary. In our case, we use this to re-endorse previous service
71 // identities, which are self-signed, and replace them with new endorsements
72 // by the current service identity (which doesn't have the private key of
73 // previous ones).
